# Multi-Agent 自动化文献综述系统设计方案

> 基于论文《AI-Augmented Literature Reviews: Efficient Clustering and Summarization for Researchers》(IEEE Access, 2025) 的方法论设计

---

## 一、问题定义

### 1.1 现有痛点

| 痛点 | 描述 |
|---|---|
| **效率低** | 研究者需手动筛选数千篇论文，耗时数周甚至数月 |
| **偏见风险** | 人工分类受主观认知影响，不同评审者分类标准不一致 |
| **覆盖不全** | 大规模文献中容易遗漏关键工作 |
| **难以复现** | 手动综述流程缺乏标准化，难以重复验证 |

### 1.2 目标

构建一个 **Multi-Agent 系统**，输入一个研究领域/主题，自动完成：
- 🔍 文献检索与去重
- 📊 相关性筛选
- 🗂️ 自动聚类与主题发现
- 📝 分主题结构化摘要生成
- 📄 输出完整文献综述报告

### 1.3 核心设计原则（源自论文）

1. **分步解耦**：将综述拆为独立阶段，每阶段由专门 Agent 负责
2. **人机协同验证**：关键节点保留人工校验接口
3. **层级研究问题驱动**：用分层 RQ 引导检索、聚类、摘要
4. **可复现**：全流程参数化、可追溯

## 三、各 Agent 详细设计

### Agent 1: Search Agent（文献检索 Agent）

**职责**：从学术数据库检索文献，去重、清洗、标准化

```
┌─────────────────────────────────┐
│        Search Agent             │
│                                 │
│  输入: 研究主题 + 搜索参数       │
│  输出: 去重后的文献池 + PDF原文   │
│                                 │
│  子步骤:                        │
│  ① 构建搜索查询                 │
│  ② 多源检索                     │
│  ③ 去重合并                     │
│  ④ 元数据标准化                 │
│  ⑤ PDF下载与文本提取            │
└─────────────────────────────────┘
```

**Prompt 设计 — 查询构建**：

```
你是一个学术文献检索专家。给定以下研究主题和范围定义，请生成
结构化的搜索策略。

研究主题：{research_topic}
范围定义：{scope_description}
目标数据库：Scopus, Google Scholar, Semantic Scholar, arXiv

请输出JSON格式：
{
  "search_queries": [
    {
      "database": "数据库名",
      "query": "搜索查询字符串",
      "filters": {
        "year_range": [起始年, 结束年],
        "language": "语言限制",
        "document_type": ["conference paper", "journal article"]
      }
    }
  ],
  "expected_coverage": "预估覆盖范围说明"
}
```

**Prompt 设计 — 去重策略**：

```
你负责对多源检索结果进行去重。给定以下两组文献标题列表，
请识别重复文献。

规则：
1. 完全相同的标题 → 直接去重
2. 标题高度相似（仅大小写/标点差异）→ 去重
3. 同一工作的预印本版和正式版 → 保留正式版

列表A（来自{source_a}）：{titles_a}
列表B（来自{source_b}）：{titles_b}

输出JSON：
{
  "duplicates": [{"title_a": "...", "title_b": "...", "reason": "..."}],
  "unique_from_a": [...],
  "unique_from_b": [...]
}
```

### Agent 2: Screen Agent（筛选 Agent）

**职责**：对文献池进行相关性筛选，替代人工初筛

```
┌─────────────────────────────────┐
│        Screen Agent             │
│                                 │
│  输入: 去重文献池 + 层级RQ       │
│  输出: 相关文献集 + 每篇相关性分数│
│                                 │
│  子步骤:                        │
│  ① 文档解析与分块               │
│  ② 向量化与索引构建              │
│  ③ 基于RQ的语义检索             │
│  ④ 归一化频率过滤               │
│  ⑤ LLM辅助相关性判定            │
└─────────────────────────────────┘
```

**核心技术流程（源自论文 Section V-A/B）**：

```
PDF文档
  │
  ▼
文本清洗（去引用/致谢/URL/特殊字符）
  │
  ▼
语义分块（按段落/句子，max_chunk_size=512, overlap=50）
  │
  ▼
Embedding编码（text-embedding-3-small, 1536维）
  │
  ▼
FAISS索引构建
  │
  ▼
按RQ检索top-k chunks
  │
  ▼
归一化频率 NF(d) = 被检索chunk数 / 总chunk数
  │
  ▼
阈值过滤（NF ≥ 0.7 → 相关）
```

**Prompt 设计 — LLM辅助相关性判定**：

```
你是一个文献筛选专家。请判断以下论文是否与研究问题相关。

研究问题（RQ）：{research_question}
论文标题：{title}
论文摘要：{abstract}
关键分块内容：
{top_chunks}

请从以下维度评估：
1. 主题相关性（1-5分）：论文是否直接回答RQ？
2. 方法相关性（1-5分）：论文方法是否属于RQ关注范围？
3. 时效性（1-5分）：论文是否反映最新进展？

输出JSON：
{
  "relevant": true/false,
  "confidence": 0.0-1.0,
  "relevance_scores": {
    "topic": 分数,
    "method": 分数,
    "timeliness": 分数
  },
  "reasoning": "判断理由（一句话）",
  "primary_theme": "论文主要贡献所属主题"
}
```

### Agent 3: Cluster Agent（聚类分析 Agent）

**职责**：将筛选后的文献按语义相似度自动聚类，发现主题结构

```
┌─────────────────────────────────┐
│        Cluster Agent            │
│                                 │
│  输入: 相关文献集 + Embedding    │
│  输出: 主题簇 + 簇标签 + 可视化  │
│                                 │
│  子步骤:                        │
│  ① 降维（t-SNE/UMAP）          │
│  ② HDBSCAN聚类                 │
│  ③ 簇标签生成                   │
│  ④ 噪声点处理                   │
│  ⑤ 聚类质量评估                 │
└─────────────────────────────────┘
```

**聚类参数配置（源自论文 Table 7）**：

```json
{
  "dimensionality_reduction": {
    "method": "t-SNE",
    "n_components": 2,
    "perplexity": 30,
    "random_state": 42
  },
  "clustering": {
    "method": "HDBSCAN",
    "min_cluster_size": 5,
    "min_samples": 3,
    "metric": "euclidean"
  }
}
```

**Prompt 设计 — 簇标签生成**：

```
你是一个文献分类专家。给定一个论文簇中的所有标题和关键摘要片段，
请为该簇生成：

1. 一个简洁的簇标签（不超过10个词）
2. 该簇的核心研究主题描述（2-3句话）
3. 该簇中的代表性论文（最多3篇）

簇内论文：
{papers_in_cluster}

输出JSON：
{
  "cluster_label": "簇标签",
  "core_theme": "核心主题描述",
  "representative_papers": [
    {"title": "...", "reason": "代表性原因"}
  ],
  "sub_themes": ["子主题1", "子主题2"]
}
```

### Agent 4: Summary Agent（综述生成 Agent）

**职责**：对每个主题簇生成结构化摘要，最终整合为完整综述报告

```
┌─────────────────────────────────┐
│        Summary Agent            │
│                                 │
│  输入: 聚类结果 + 层级RQ         │
│  输出: 分主题综述 + 整体综述报告  │
│                                 │
│  子步骤:                        │
│  ① 按簇提取论文内容             │
│  ② RAG摘要生成（二级RQ）        │
│  ③ 深度分析（三级RQ）           │
│  ④ 跨簇趋势综合                 │
│  ⑤ 生成最终报告                 │
└─────────────────────────────────┘
```

**Prompt 设计 — 二级RQ摘要（方法论维度）**：

```
你是一个学术综述撰写专家。请基于以下论文簇的内容，回答研究问题。

研究问题（RQ11）：该簇中的论文使用了哪些方法论？

簇标签：{cluster_label}
簇内论文内容（RAG检索结果）：
{rag_retrieved_content}

请按以下结构输出：
1. 【方法概述】该簇涉及的主要方法类型（列表）
2. 【技术细节】每种方法的核心技术要点（2-3句/方法）
3. 【方法对比】各方法的优势和局限（表格形式）
4. 【演进趋势】方法随时间的变化趋势
5. 【研究空白】当前方法尚未解决的问题
```

**Prompt 设计 — 二级RQ摘要（应用维度）**：

```
研究问题（RQ21）：这些方法被应用在哪些领域？

簇内论文内容：
{rag_retrieved_content}

请按以下结构输出：
1. 【应用领域列表】该簇论文涉及的应用领域
2. 【领域-方法映射】每个领域使用的主要方法
3. 【领域挑战】每个领域面临的主要挑战和局限
4. 【跨领域机会】不同领域之间可能的交叉机会
```

**Prompt 设计 — 最终综述整合**：

```
你是一个资深学术综述撰写专家。基于以下各主题簇的分析结果，
请撰写一份完整的文献综述报告。

研究主题：{research_topic}
综述时间范围：{year_range}
分析论文总数：{total_papers}

各簇摘要：
{cluster_summaries}

请按以下结构撰写（中文，学术风格）：

1. 引言
   - 研究背景与意义
   - 综述范围与方法论说明

2. 方法论综述
   2.1 [Cluster A标签]
   2.2 [Cluster B标签]
   ...
   （每节包括：方法原理、代表性工作、优缺点对比）

3. 应用领域综述
   3.1 [领域A]
   3.2 [领域B]
   ...
   （每节包括：应用场景、技术方案、效果评估）

4. 研究趋势与演进
   - 方法论演进时间线
   - 应用领域扩展趋势
   - 跨学科融合趋势

5. 挑战与未来方向
   - 当前研究的共性局限
   - 未来可能的突破方向
   - 对从业者的建议

6. 结论

要求：
- 每个论点需引用具体论文支撑
- 对比分析要客观、有数据支撑
- 语言精炼，避免空洞表述
```

## 五、层级 RQ 设计模板

这是整个系统的**核心驱动力**——所有 Agent 的工作都围绕层级研究问题展开：

```
                        ┌─────────────┐
                        │  Research   │
                        │  Topic      │
                        └──────┬──────┘
                               │
                  ┌────────────┴────────────┐
                  ▼                         ▼
          ┌──────────────┐         ┌──────────────┐
          │   RQ1        │         │   RQ2        │
          │  方法论维度   │         │  应用维度     │
          └──────┬───────┘         └──────┬───────┘
                 │                        │
         ┌───────┴───────┐        ┌───────┴───────┐
         ▼               ▼        ▼               ▼
    ┌─────────┐   ┌─────────┐ ┌─────────┐  ┌─────────┐
    │  RQ11   │   │  RQ12   │ │  RQ21   │  │  RQ22   │
    │用了哪些 │   │优缺点是 │ │应用在   │  │各领域   │
    │方法？   │   │什么？   │ │哪些领域？│  │的挑战？  │
    └────┬────┘   └────┬────┘ └────┬────┘  └────┬────┘
         │              │          │             │
         ▼              ▼          ▼             ▼
    ┌─────────┐   ┌─────────┐
    │  RQ111  │   │  RQ121  │
    │时间依赖 │   │方法具体 │
    │如何表示？│   │优势？   │
    └─────────┘   └─────────┘
```

**Prompt — 自动生成层级 RQ**：

```
你是一个研究方法论专家。给定研究主题，请生成层级式研究问题，
用于指导自动化文献综述。

研究主题：{research_topic}
综述目标：{review_objective}

请生成3层研究问题：
- Level 1（2-3个）：宏观维度（如方法论、应用、趋势）
- Level 2（每个L1下2-3个）：中观分析（如具体方法类型、具体领域）
- Level 3（每个L2下1-2个）：微观深入（如技术细节、具体优缺点）

输出JSON：
{
  "research_questions": [
    {
      "id": "RQ1",
      "level": 1,
      "question": "...",
      "children": [
        {
          "id": "RQ11",
          "level": 2,
          "question": "...",

## 六、质量评估体系

### 6.1 各阶段评估指标

| 阶段 | 指标 | 计算方式 | 目标值 |
|---|---|---|---|
| Search | 覆盖率 | 系统检索数 / 数据库实际相关数 | ≥90% |
| Screen | 检索准确率 | 正确筛出数 / 人工标注总数 | ≥85% |
| Screen | 精确率 | 正确筛出数 / 系统筛出总数 | ≥80% |
| Cluster | 调整兰德指数 | 与人工分类的一致性 | ≥0.7 |
| Cluster | 轮廓系数 | 簇内紧密度 vs 簇间分离度 | ≥0.5 |
| Summary | ROUGE-L | 与人工摘要的重叠度 | ≥0.6 |
| Summary | 事实一致性 | LLM评估摘要是否忠实于原文 | ≥90% |

### 6.2 端到端评估 Prompt

```

## 八、与论文方法的关键对应

| 论文步骤 | 对应 Agent | 增强 |
|---|---|---|
| Stage 1-3（定义范围+搜索） | Search Agent | +多源API自动化，+LLM辅助查询构建 |
| Stage 4-6（筛选+质量评估） | Screen Agent | +归一化频率+LLM判定双保险 |
| Stage 7-8（主题识别+分类） | Cluster Agent | 完全继承论文的t-SNE+HDBSCAN方案 |
| Stage 9（分析总结） | Summary Agent | +层级RQ驱动的RAG摘要 |
| 全程人工验证 | Coordinator Agent | 4个质量门控节点 |

---

**总结**：这个设计的核心思想是**用层级RQ作为全局线索串联4个专业Agent**，每个Agent只做一件事但做到极致——Search负责广度覆盖，Screen负责精度筛选，Cluster负责结构发现，Summary负责深度理解。Coordinator在关键节点插入人工审核，确保"AI加速但不失控"。整体严格基于论文验证过的方法论，但通过Multi-Agent架构实现了更高的自动化程度和模块化可扩展性。
