# Multi-Agent 文献综述系统 — 项目技术报告

## 项目概述

设计并实现了一套基于多Agent协作的自动化文献综述系统。系统接收任意研究主题，通过检索、筛选、聚类、综述四阶段流水线，自动生成包含底层原理分析、量化对比和精确引用的结构化综述报告。

## 系统架构

采用 Coordinator-Agent 协作架构，核心由 `Coordinator`（协调者）编排四个专业 Agent 顺序执行，通过 `SharedWorkspace`（共享工作区）实现跨 Agent 数据持久化与状态共享。协调者内置四级质量门控（Quality Gate）和检查点回滚机制，支持流程中断恢复。

每个 Agent 继承自 `BaseAgent` 抽象基类，遵循 Template Method 模式：基类封装 LLM 调用（全局 asyncio.Semaphore(5) 信号量限流）、结果标准化、错误处理；子类实现 `execute()` 定义具体逻辑。

## 核心技术实现

### 1. 多源异构文献检索

`SearchAgent` 通过 LLM 自动生成领域限定的搜索策略，回退到关键词组合兜底。集成 arXiv、PubMed、DBLP、Europe PMC、OpenAlex 五个免费学术数据库 API，统一异步接口（aiohttp），基于标题标准化去重，输出标准化 `LiteratureRecord` 数据结构。

### 2. 三级相关性筛选

`ScreenAgent` 实现了三级递进筛选：

- **语义相似度初筛**：将全部论文标题+摘要与层级研究问题（RQ）分别通过 BGE/all-MiniLM 嵌入，计算余弦相似度矩阵（numpy 向量化），每篇论文取跨所有 RQ 的最大相似度作为初始分数。
- **分层决策**：≥0.80 自动通过、<0.68 自动拒绝、[0.68, 0.80) 边界区间送 LLM 逐篇判定（最多150篇），LLM 从主题相关性、方法论相关性、时效性三维度评分。
- **嵌入模型自适应**：根据论文内容语言自动选择 local-zh（BGE-small-zh-v1.5）或 local-en（all-MiniLM-L6-v2），加载失败自动回退。

### 3. GPU 加速语义聚类

`ClusterAgent` 仅对通过筛选的相关论文进行聚类：

- **降维**：两步降维策略。高维→50维通过 PCA（PyTorch CUDA 加速，CPU 回退），50维→2维根据样本量选择 PCA（>1000样本）或 t-SNE（≤1000样本）。
- **聚类**：HDBSCAN 为主算法（自适应 min_cluster_size：论文少时自动缩小），KMeans 和距离阈值层次聚类作为回退方案。
- **后处理**：LLM 并行生成簇标签，基于主题关键词的领域一致性过滤移除不相关簇（匹配率 < 30% 移除）。

### 4. 反幻觉综述生成

`SummaryAgent` 采用两段式 LLM 调用生成完整报告（突破单次 token 限制）：

- **Part 1**：研究背景、核心挑战、分簇技术原理综述（要求覆盖全部簇）
- **Part 2**：技术效果量化分析、代际演进逻辑、物理极限分析、突破路径建议

提示词工程的核心设计：
- **反幻觉引用**：强制 `[作者, 年份, 论文标题]` 格式，提供正确/错误示例对比，禁止引用论文列表外的文献
- **反空泛表述**：禁止"取得了较好效果"类评价，必须给出具体指标、数值变化和底层原因
- **深度分析约束**：要求深入物理机制/数学公式层面，每个结论必须能追溯到具体原理
- **兜底机制**：始终生成模板 fallback 报告，LLM 失败时自动降级

### 5. 层级研究问题（RQ）管理

`RQManager` 为每个研究主题自动生成三级 RQ 树结构（方法论维度 / 应用维度 / 趋势维度），每个层级的问题均包含完整的研究主题上下文，确保下游筛选和综述阶段的语义匹配精度。RQ 树驱动整个筛选-聚类-综述流程。

## 性能优化

| 优化项 | 实现方式 |
|--------|----------|
| LLM 并发控制 | 全局 asyncio.Semaphore(5)，所有 LLM 调用通过基类统一限流 |
| GPU 加速 | PyTorch CUDA PCA 降维，FAISS 向量索引（GPU > CPU > NumPy 回退链） |
| 批量更新 | 筛选阶段内存批量计算，单次 JSON 持久化（避免逐条写入） |
| LLM 客户端缓存 | 按模型+参数组合缓存 ChatOpenAI 实例，避免重复初始化 |
| 嵌入缓存 | 聚类阶段复用已缓存 embedding，仅补充缺失部分 |
| 异步并行 | 簇摘要并行生成（asyncio.gather）、数据库查询异步非阻塞 |

## 技术栈

Python 3.10+ · LangChain · asyncio/aiohttp · PyTorch (CUDA) · HDBSCAN · scikit-learn · NumPy · BGE/all-MiniLM 嵌入模型 · FAISS · YAML 配置 · JSON 持久化

## 工程实践

- 分层配置体系：YAML 全局配置 + 环境变量覆盖 + Agent 级参数（各 Agent 独立 max_tokens、temperature）
- 结构化日志：每阶段输出论文数量、分数分布、耗时等关键指标
- 检查点机制：`SharedWorkspace` 支持创建/恢复检查点，流程中断可回滚
- 完整测试套件：单元测试 + 集成测试，覆盖工作区 CRUD、RQ 管理、聚类流程
